**Format:** Markdown blueprint for design, content, UX, and implementation  
**Prepared for:** EncouragingYou

## 1. Project North Star

EncouragingYou should feel like a **trusted digital front door** for young people and community members who need support, confidence, connection, and practical next steps.

The website should not feel like a generic charity brochure. It should feel:

- warm
- credible
- youth-led
- welcoming
- practical
- community-rooted
- hopeful without being sentimental
- modern without being trendy for trend’s sake

The site’s job is to help four audience groups act with confidence:

1. **Young people** who want support, confidence, community, mentoring, or help with CVs and opportunities.
2. **Parents, carers, schools, and referrers** who need reassurance, clarity, and a safe route to support.
3. **Wider community members**, including older or isolated people who may need connection or practical help.
4. **Volunteers, partners, and funders** who need to see a real, growing organisation with clear purpose and visible trust signals.

### Strategic website objective

Turn interest into action by making it easy to:

- join a session
- get support
- understand what EncouragingYou does
- trust the organisation
- volunteer or partner
- stay informed about events and updates

## 2. Brand Experience Summary

## 2.1 Brand essence

**Encouragement with credibility.**

EncouragingYou exists to help people feel they belong, build confidence, gain practical skills, and move forward.

## 2.2 Emotional tone

The site should feel:

- **Supportive** rather than institutional
- **Clear** rather than wordy
- **Human** rather than polished-corporate
- **Safe** rather than formal
- **Hopeful** rather than pity-led
- **Dignified** rather than patronising

## 2.3 Personality sliders

| Trait | Lean toward |
|---|---|
| Formal ↔ Friendly | Friendly |
| Corporate ↔ Human | Human |
| Serious ↔ Uplifting | Uplifting with seriousness where needed |
| Youthful ↔ Childish | Youthful, never childish |
| Inspirational ↔ Practical | Practical first, inspirational second |
| Busy ↔ Calm | Calm |
| Glossy ↔ Grounded | Grounded |

## 2.4 What the brand should avoid

Avoid a look and feel that is:

- generic charity stock-site
- pity-based or deficit-framed
- overly loud or neon
- cluttered with too many CTAs
- too child-focused
- too sleek or startup-like
- heavy with jargon
- built around fake “impact” imagery

## 3. Visual Direction

## 3.1 Overall visual thesis

The visual system should combine:

- **warm editorial illustration**
- **clear community-centre realism**
- **modern modular UI**
- **soft, inclusive surfaces**
- **clean iconography**
- **high readability**
- **low visual stress**

The attached AI illustrations already point to the right direction: warm light, grounded interiors, contemporary community spaces, diverse people, and a calm feeling of activity. The design system should support those images rather than compete with them.

## 3.2 Colour palette

The palette should be drawn from the illustration and icon suite.

### Core palette

| Token | Hex | Role |
|---|---:|---|
| Ink / Deep Teal | `#00405C` | Primary text, primary buttons, icons, key UI accents |
| Warm Coral | `#FC8C6C` | Accent, highlights, chips, icon secondary colour, subtle emphasis |
| Soft Sage | `#90B4A0` | Secondary accent, reassurance blocks, supportive UI highlights |
| Warm Cream | `#F8E8D0` | Main section background, hero surfaces, soft cards |
| Sand | `#E6D0B0` | Alternate background, dividers, quiet panels |
| Chalk | `#F5F3EF` | Neutral page background |
| Charcoal | `#1F2224` | High-contrast headings where needed |
| White | `#FFFFFF` | Cards, forms, content panels |

### Recommended usage rules

- Use **Deep Teal** for core text, buttons, navigation, links, and icon strokes.
- Use **Warm Coral** as a highlight colour, not as the primary body-text colour.
- Use **Warm Cream** and **Chalk** as the dominant backgrounds so the site feels soft and open.
- Use **Soft Sage** in trust panels, wellbeing cues, and calm supporting sections.
- Keep bright colour usage controlled; the site should feel warm and reassuring, not loud.

### Accessibility rule

- Do not place white text on Warm Coral unless the shade is darkened for contrast.
- Default body text should sit on white, chalk, or warm cream backgrounds using Deep Teal or Charcoal.
- Primary buttons should use **Deep Teal** with white text.
- Secondary buttons should use white or warm-cream backgrounds with Deep Teal text and border.

## 3.3 Typography

Use a typography system that feels human and modern.

### Recommended pairing

- **Primary UI / body / navigation / buttons:** `Inter` variable
- **Editorial accent / mission pull quotes / selected page headings:** `Newsreader`

### Why this works

- `Inter` keeps forms, labels, body copy, and navigation highly legible.
- `Newsreader` adds a warm editorial note without becoming luxurious or stiff.
- If a single-font system is preferred, use `Inter` only.

### Typographic scale

| Use | Desktop | Mobile |
|---|---:|---:|
| H1 | 56–64px | 36–42px |
| H2 | 40–48px | 28–32px |
| H3 | 28–32px | 22–24px |
| Body Large | 20–22px | 18px |
| Body | 18px | 16–18px |
| Small / meta | 14–16px | 14–16px |

### Typographic behaviour

- Keep line length between 60 and 75 characters in text-heavy blocks.
- Use strong heading hierarchy.
- Keep paragraphs short.
- Prefer sentence case over title case for a more human feel.
- Use bold sparingly and meaningfully.

## 3.4 Layout and spacing

### Grid

- **Desktop:** 12-column grid, max content width 1200px
- **Tablet:** 6-column
- **Mobile:** 4-column or fluid single-column
- **Section gutters:** generous; do not cram content edge to edge

### Spacing rhythm

| Token | Value |
|---|---:|
| XS | 8px |
| S | 12px |
| M | 16px |
| L | 24px |
| XL | 32px |
| 2XL | 48px |
| 3XL | 72px |
| 4XL | 96px |

### Shape language

- Rounded corners: **20px to 24px** on major cards and panels
- Softer 12px radius on small UI elements
- Avoid hard, sharp boxes
- Use subtle shadows and layered surfaces, not heavy floating-card effects

## 3.5 Texture and background treatment

Use very light texture only where helpful:

- subtle paper-like softness
- faint line-based motifs derived from icons
- soft section colour shifts
- occasional curved dividers or path motifs

Avoid:

- noisy patterns
- gradient overload
- abstract blobs with no meaning
- trendy dark-mode-only aesthetics

## 3.6 Iconography

The attached icon suite is strong and aligned to the brand.

### Icon style rules

- Rounded line icons
- Deep Teal primary stroke
- Warm Coral highlights
- Simple, readable silhouettes
- Use consistently at 48px, 64px, or 80px sizes
- Export each icon separately for production use

### Recommended icon mapping

| Meaning | Icon |
|---|---|
| Safe space / welcome | Open doorway with heart |
| Mentoring / conversation | Two people speaking |
| CV support / opportunity | CV/document with upward arrow |
| Youth activities | Group activity icon |
| Confidence / growth | Pathway / star / steps |
| Community event | Tent / gathering |
| Intergenerational support | Two people at table |
| Contact / support line | Phone with heart |

### Production note

If possible, rebuild the icons as SVGs from the current artwork so they stay crisp at all screen sizes. If not, export transparent PNGs at 2x and 3x.

## 3.7 Motion

Motion should be subtle and non-essential.

### Recommended

- soft fade-in or rise-in on scroll for cards
- gentle hover state on cards and buttons
- sticky header transition on scroll
- no major animated hero effects
- respect `prefers-reduced-motion`

### Avoid

- autoplay video
- parallax
- fast sliders
- bouncing UI
- rotating carousels
- animated counters on load

## 5. Experience Principles

These principles should guide every page and component.

## 5.1 Lead with user needs

Users should instantly understand:

- what EncouragingYou does
- whether the service is for them
- what happens next
- how to get help

## 5.2 Make first steps easy

The best website for this organisation is not the one with the most pages. It is the one that reduces hesitation.

Every major page should answer:

- Is this for me?
- What happens?
- Is it safe?
- How do I get started?

## 5.3 Show warmth through clarity

Warmth here comes from:

- plain-English copy
- reassuring structure
- visible people-centred imagery
- clear paths to action
- dignified language

## 5.4 Build trust visibly

Trust should not live only on legal pages. It should show up in:

- safeguarding access
- volunteer and staff checks
- who runs the organisation
- session details
- contact options
- privacy clarity
- honest image labelling

## 5.5 Keep the site lightweight

- No hero carousel
- No heavy social embeds on page load
- No autoplay media
- No unnecessary popups
- No hidden core information

**Home → Programme card / quick action → Programme detail → Join / Contact**

- Is this welcoming?
- What happens at the session?

**Home / Get Involved → Volunteer or Partner page → Enquiry form**

Key questions:

- Is there a real role?

## 9. Global UI Patterns

## 9.1 Header

### Desktop
- logo left
- navigation centre or right
- standout CTA button on the far right (`Join a session`)

### Mobile
- logo left
- menu toggle right
- optional sticky bottom CTA for `Get support` if testing shows it helps

### Header behaviour
- sticky after scroll
- solid or lightly translucent background
- never fully transparent over busy imagery

## 9.2 Navigation labels

Recommended top-level labels:

- Home
- About
- Programmes
- Events & Updates
- Get Involved
- Safeguarding
- Contact

## 9.3 Buttons

### Primary
- Deep Teal fill
- white text
- rounded corners
- medium weight
- subtle hover lift

### Secondary
- white or warm-cream surface
- Deep Teal border and text
- subtle hover surface shift

### Tertiary / text links
- Deep Teal text
- subtle coral underline or icon

## 9.4 Cards

Card families should feel related across the site.

### Card rules
- soft radius
- image first or icon first
- consistent internal spacing
- clear hover state
- never overloaded with too much text

## 9.5 Forms

Form style should feel calm and supportive:

- white background
- 1-column layout
- clear labels above fields
- no placeholder-only labelling
- helpful error messages
- good focus states
- explicit privacy note

## 9.6 Banners and notices

Use only for truly important items such as:
- urgent service update
- venue change
- event date change

Do not fill the top of the site with low-priority announcements.

## 9.7 FAQ pattern

Use progressive disclosure:
- native `<details>` / `<summary>` or accessible accordion
- keep answers short
- only include questions users are likely to ask

## 9.8 AI disclosure pattern

Use a small caption or chip under illustrations:

- `AI-generated illustration — not a photograph of participants`
- `Illustrative image for launch`

Use on image-heavy contexts, not on every single decorative instance if it becomes repetitive.

## 12. Technical Architecture Recommendation

## 12.1 Recommended build approach

Use a **static-first, CMS-driven website** with progressive enhancement.

### Recommended stack

- **Front end:** Astro or Next.js in static-first mode
- **CMS:** a simple headless CMS such as Sanity, Storyblok, Directus, or a lightweight structured content backend
- **Hosting:** modern edge/static hosting
- **Forms:** lightweight server-side or serverless processing with explicit privacy handling
- **Analytics:** privacy-conscious and minimal
- **Search:** not required for launch unless content volume grows

### Why this fits

- content-heavy but not app-heavy
- performance matters
- content needs to be editable
- future expansion is likely
- repeatable page templates are important

## 14. Performance, Accessibility, and SEO Standards

## 14.1 Accessibility baseline

Build to WCAG 2.2 AA as the minimum standard.

### Key implications for this site

- text contrast must be strong
- focus states must be clearly visible
- forms must have proper labels
- target sizes must be comfortable on mobile
- accordions must be keyboard-friendly
- motion must respect reduced-motion preferences
- decorative images should use empty alt text
- informative illustrations should have meaningful alt text

## 14.2 Performance rules

- Export attached PNG illustrations to responsive AVIF/WebP for production
- Do not lazy-load the above-the-fold hero image
- Lazy-load lower-page illustrations and non-critical images
- Minimise JavaScript
- Avoid heavy third-party embeds
- Keep font loading simple
- Serve images at realistic sizes only

### Core targets

- LCP under 2.5s
- INP under 200ms
- CLS under 0.1

## 14.3 SEO baseline

- Clear page titles and meta descriptions
- Structured data for organization and events
- Descriptive headings
- Strong internal linking
- Human alt text
- One clear topic per page
- Service area mentioned naturally in content

### Suggested title patterns

- Home: `EncouragingYou | Youth-led community support in Rochdale`
- About: `About EncouragingYou | Youth-led community support`
- Programme pages: `[Programme Name] | EncouragingYou`
- Contact: `Contact EncouragingYou`

## 18. Final Creative Direction Summary

If this website is executed well, it should feel like this:

> A warm, modern, youth-led community website that makes people feel welcome within seconds, explains support clearly, reduces anxiety, and turns intention into confident action.

It should look like:

- warm light
- soft surfaces
- clear structure
- real community energy
- calm confidence
- clean line icons
- strong accessibility
- simple pathways into support

It should not look like:

- a generic charity template
- an overbuilt corporate site
- a child-oriented activity club
- a content-heavy PDF dump
- an AI-generated fantasy world

The attached illustration suite is already strong enough to support this direction for launch. The job of the site design is to make those visuals feel purposeful, ethical, and usable — not just decorative.
