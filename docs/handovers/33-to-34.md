# Prompt 33 To 34 Handover

Prompt 33 replaced the old `mailto` enquiry transport with a secure shared server path.

## What is now stable

- all shared enquiry surfaces now post to `site/src/pages/api/enquiry.ts`
- canonical normalization and validation now live in `site/src/lib/state/support-form.js`
- anti-abuse and queue storage now live in `site/src/lib/server/enquiry-service.js`
- shared route/context/query rules now live in `site/src/lib/forms/enquiry-contract.js`
- user-facing fallback copy now lives in `site/src/lib/forms/enquiry-feedback.js`
- session detail `Ask to join` CTAs now route into Contact with validated context instead of `mailto:`
- Home, Contact, Get Involved, Volunteer, and Partner all use the same secure form transport

## What Prompt 34 should do

- keep safeguarding concern handling separate from the ordinary enquiry queue
- treat Prompt 33's `support-general` surface as general intake only, not as a concern-report form
- use the new secure-form architecture only where it is operationally appropriate for safeguarding
- decide whether safeguarding needs its own dedicated intake surface, route split, or both

## Watchouts

- do not reintroduce `mailto:` as the primary form transport
- do not let safeguarding inherit the normal volunteer/partner/general success copy
- do not bypass the new origin/context/allowlist checks when adding safeguarding-specific handling
- do not remove the plain email fallback unless a stronger verified channel exists
