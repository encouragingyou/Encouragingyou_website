# Handover: Prompt 07 to Prompt 08

Prompt 07 is complete.

## What changed

- Added workspace governance files:
  - repo-root `.editorconfig`
  - repo-root `README.md`
  - `site/.gitignore`
  - `site/.npmrc`
  - `site/.nvmrc`
  - `site/.prettierrc.json`
  - `site/.prettierignore`
  - `site/eslint.config.mjs`
- Added environment templates:
  - `site/.env.example`
  - `site/.env.preview.example`
- Expanded `site/package.json` with lint, format, validate, CI, and advisory audit commands.
- Added Node/npm engine declarations and package-manager pinning.
- Made `SITE_URL` configurable in `site/astro.config.mjs`.
- Added repo CI at `.github/workflows/site-quality.yml`.
- Added developer docs:
  - `docs/developer/07-workspace-topology.md`
  - `docs/developer/07-tooling-and-command-surface.md`
  - `docs/developer/07-environment-matrix.md`
  - `docs/developer/07-ci-quality-gates.md`

## What is now proven

- Contributors no longer need `--workspaces=false` to use npm in `site/`.
- The repo has explicit lint, format, type/content, unit, build, and browser QA commands.
- CI has a deterministic blocking pipeline plus a separate advisory audit.
- Environment expectations are documented without inventing secrets.

## Known limitations

- Several runtime modules still remain in `*.js`; the standard is now "TypeScript for new shared logic, migrate touched JS opportunistically".
- Bridged legacy HTML and copied public assets are still temporary and intentionally ignored by formatter/lint tooling.
- CI currently relies on the built preview path for Playwright rather than a deployment preview platform.
- `npm run audit:advisory` currently reports 6 moderate findings tied to Astro 5 and checker transitive dependencies; clearing them requires a planned breaking-version upgrade rather than a prompt-07 hotfix.

Prompt 08 should build the token system directly on top of the now-formalized styles layer and should keep the new lint, check, build, and Playwright gates green.
