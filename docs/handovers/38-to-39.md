# Prompt 38 To 39 Handover

Prompt 38 makes AI-illustration honesty part of the production system and adds a dedicated legal/disclosure Playwright contract.

## What is now stable

- `site/src/content/mediaLibrary/default.json` now carries source type, evidence status, consent status, safeguarding sensitivity, disclosure rules, replacement targets, and restricted route families
- `site/src/content/notices/default.json` now carries the canonical prominent/compact/sitewide AI-illustration text variants
- `site/src/lib/content/media-disclosure.ts` is the canonical disclosure resolver
- `site/src/components/ui/DisclosureNote.astro` is the shared disclosure renderer with stable testing attributes
- `site/scripts/validate-media-library.mjs` now enforces disclosure-variant presence and AI-media truth rules
- `npm run test:e2e:trust --workspaces=false` is the focused contract for legal, consent, disclosure, and responsive trust checks

## What Prompt 39 should assume

- disclosure markup and selector attributes are now first-class public-site infrastructure
- footer-wide AI-illustration honesty is canonical and should not be re-hardcoded in route templates or metadata helpers
- synthetic imagery is intentionally blocked from proof-bearing route families unless real approved assets arrive
- the trust suite can be extended for metadata/SEO regressions, but should not be replaced

## Watchouts

- do not add SEO metadata that implies documentary photography where the route uses AI illustration
- do not create social-preview or schema-image logic that bypasses the media truth model
- if Prompt 39 introduces image-based metadata for proof/trust surfaces, it must respect `restrictedRouteFamilies`, `evidenceUse`, and `consentStatus`
