# Prompt 12 To 13 Handoff

Prompt 12 established the reusable component contract that Prompt 13 should treat as stable unless real content-loading evidence proves it insufficient.

## What is now in place

- Shared primitives for buttons, badges, action rows, cards/panels, notices, accordions, fields, checkboxes, and CTA bands
- Composed aliases rebuilt on top of those primitives: `ActionCard`, `SessionSummaryCard`, `ProgrammeCard`, `FaqGroup`, `SupportForm`
- Representative route refactors across About, Privacy, Safeguarding, Sessions, Programme detail, Get Involved, and the site footer
- Internal preview route at `/system/components/` for deterministic component-state validation
- Playwright coverage for component states, shell focus behavior, nested current-route handling, and form recovery

## Prompt 13 focus

Prompt 13 should now move hardcoded page composition and copy dependencies further into structured content sources without reworking the component APIs first.

Priority areas:

- centralize more route copy and panel content currently assembled in `site-content.ts`
- keep `/system/components/` as a fixture route; it can stay hardcoded because it is a QA harness, not editorial content
- preserve the parent-nav current-path behavior for nested placeholder routes unless Prompt 13 replaces those placeholders with real content pages

## Watch-outs

- Do not reintroduce bespoke `article` markup where `CardPanel` already fits.
- Do not bypass `FormField` or `ActionCluster` for new route sections.
- If Prompt 13 adds new content-driven sections, prefer extending structured data models before adding new component variants.
