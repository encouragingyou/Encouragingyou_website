# Prompt 35 To 36 Handover

Prompt 35 turns `/privacy/` into a real production notice tied to the implemented enquiry stack.

## What is now stable

- `site/src/content/privacyNotice/default.json` is the canonical privacy truth source
- `site/src/pages/privacy/index.astro` is a real production page with a table of contents, live collection-point cards, system-state callouts, rights guidance, and change-control triggers
- `site/src/components/ui/PrivacyNoticeCallout.astro` is the shared point-of-collection component
- every live `SupportForm` route now receives `privacyHighlights` from `site/src/content/formSurfaces/default.json`
- `site/scripts/validate-structured-content.mjs` now checks privacy collection-point coverage and launch-state flags

## What Prompt 36 should do

- keep the cookie policy equally narrow and truthful
- use `privacyNotice.settings` as the current-state reference before adding any consent logic
- only introduce a banner or preferences state if there is a real non-essential cookie or equivalent technology to explain
- update the privacy notice if Prompt 36 changes analytics, cookies, or processor states

## Watchouts

- do not invent analytics, cookies, or third-party processors just to justify a banner
- do not break the new `privacy-change-control-required` governance rule in `site/src/content/launchGovernance/default.json`
- do not move point-of-collection privacy text back into route-local hardcoded prose
- if Prompt 36 adds consent storage, it must be reflected in both `site/src/content/privacyNotice/default.json` and the cookie notice content
