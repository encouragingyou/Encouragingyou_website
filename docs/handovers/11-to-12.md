# Handover: Prompt 11 to Prompt 12

Prompt 11 is complete.

## What changed

- replaced the old thin shell wrapper with a route-aware shell model in `site/src/lib/content/site-shell.ts`
- added shell configuration data in `site/src/content/shellConfig/default.json`
- added shared shell primitives:
  - `site/src/components/site/SiteSkipLinks.astro`
  - `site/src/components/site/ShellWayfinding.astro`
  - `site/src/components/site/SiteNoticeStack.astro`
- rebuilt `SiteHeader.astro`, `SiteFooter.astro`, and `BaseLayout.astro` around contextual CTA, utility navigation, breadcrumb, back-link, and footer-support behavior
- migrated `/about/` and `/privacy/` into Astro-owned routes
- removed `/about/` and `/privacy/` from the legacy bridge manifest
- standardized the support-form privacy reminder through the shared notice component
- expanded Playwright with `tests/e2e/contracts/shell-wayfinding.spec.mjs`

## What is now proven

- launch route classes share one shell contract instead of mixed Astro and legacy header/footer behavior
- breadcrumbs and back-links are generated centrally
- mobile navigation still works with JS, and primary navigation still works without JS
- contextual header CTA switching is protected by tests
- `/about/` and `/privacy/` no longer depend on wrapped prototype HTML
