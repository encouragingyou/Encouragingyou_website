# Prompt 13 To 14 Handoff

Prompt 13 completes the structured content runtime baseline.

## What is now in place

- New page-composition collections in `site/src/content/formSurfaces`, `routePages`, `sessionPageContent`, and `programmePageContent`
- A stable adapter boundary in `site/src/lib/content/content-source-adapter.ts`
- Route-model composition in `site/src/lib/content/site-content.ts` with explicit required-content guards
- Astro routes refactored so public copy comes from structured content instead of route-local strings
- Build-time validation in `site/scripts/validate-structured-content.mjs`, wired into `npm run validate` and `npm run ci`

## Prompt 14 focus

Prompt 14 should treat the new content runtime as stable and extend the session domain rather than reworking route copy again.

- Keep using `sessions/*.json` as the canonical recurring-session fact layer.
- Keep `sessionPageContent` limited to editorial/presentation copy.
- Build any new recurring-session, calendar, or structured-event outputs on top of the existing `session-runtime.ts` and `session-schedule.js` helpers.

## Watch-outs

- Do not move public-facing route copy back into Astro files.
- Do not bypass the adapter just because the content is still local JSON.
- If Prompt 14 adds new session-related structured data, update `validate-structured-content.mjs` instead of relying on manual review.
