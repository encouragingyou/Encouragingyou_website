# Handover: Prompt 10 to Prompt 11

Prompt 10 is complete.

## What changed

- replaced prototype-derived image/icon copying with a canonical media pipeline
- expanded `site/src/content/mediaLibrary/default.json` into a real media manifest with provenance, delivery, alt/disclosure, and replacement metadata
- added `site/src/lib/media/catalog.ts` as the runtime bridge from asset IDs to Astro-imported local masters
- added shared media abstractions:
  - `site/src/components/ui/MediaPicture.astro`
  - `site/src/components/ui/MediaIcon.astro`
- added build-time media tooling:
  - `site/scripts/sync-media-assets.mjs`
  - `site/scripts/validate-media-library.mjs`
- regenerated compatibility outputs in `site/public/images/` and `site/public/icons/` from the canonical masters instead of copying `source/blurpint/images` and `source/blurpint/icons`
- updated the imported prototype session snapshot so it now includes `imageMediaId`

## What is now proven

- production routes can consume media by ID through structured metadata and shared components
- Astro routes now use local imported masters for responsive illustration delivery
- wrapped legacy routes still work because compatibility files are generated from the same canonical masters
- the repo has automated checks for missing masters, broken compatibility renders, alt/disclosure mismatch, and stray direct asset-path usage

## Constraints prompt 11 should keep

- do not reintroduce hardcoded `/images/...` or `/icons/...` references in Astro routes
- route shell work should keep using asset IDs and shared media components rather than direct imports inside many page files
- `/about/` and `/privacy/` are still legacy wrappers, so compatibility outputs must remain intact until those routes are migrated
- icon vectorization is intentionally deferred; do not treat the raster launch masters as a bug unless prompt scope explicitly includes icon redesign

## Known limitations

- icons still ship from raster masters because the approved source set is PNG-only
- AI illustration disclosure exists, but later prompts still need to decide where disclosure should become more prominent in editorial layouts
- trust-critical real-photography replacement is modeled in metadata, not implemented yet
