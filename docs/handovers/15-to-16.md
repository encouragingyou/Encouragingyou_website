# Prompt 15 To 16 Handoff

Prompt 15 establishes the editorial operating system for the site.

## What is now in place

- canonical editorial governance in `site/src/content/editorialSystem/default.json`
- runtime helpers in `site/src/lib/content/editorial-guidance.ts`
- shared support-form microcopy now sourced from the editorial system
- shared placeholder/status governance now rendered through `site/src/components/site/LaunchPlaceholder.astro`
- project-jargon linting in `site/scripts/validate-editorial-content.mjs`, wired into `npm run validate`
- cleaned public copy in `routePages/default.json` and `programmePageContent/default.json` so launch routes stop talking about migration internals

## Prompt 16 focus

Prompt 16 should use the new editorial system as a hard constraint when reworking homepage strategy.

- treat `editorialSystem.contentPatterns` as the approved pattern library
- use `editorialSystem.templateContracts` and `futurePromptChecklist` before landing homepage copy changes
- keep shared support-form and placeholder microcopy centralized instead of reintroducing local strings

## Watch-outs

- do not reintroduce build-process language such as prompts, runtimes, migrations, or tooling into public content
- do not turn homepage trust copy into generic charity slogans
- do not bypass the editorial validator by moving public copy into route files or components
- if prompt 16 adds new public content patterns, extend `editorialSystem/default.json` and `validate-editorial-content.mjs` together
