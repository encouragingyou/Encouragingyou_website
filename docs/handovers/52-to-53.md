# Prompt 52 to Prompt 53 Handover

Prompt 52 is now the contract layer for the CMS extension.

Use these files as the starting point for Prompt 53:

- `site/src/content/cmsScope/default.json`
- `site/src/content/cmsWorkflow/default.json`
- `site/src/lib/cms/state-machine.js`
- `site/src/lib/cms/content-model.js`
- `site/src/lib/cms/public-read-model.js`
- `site/src/data/generated/cms-schema-catalog.json`
- `site/src/data/generated/cms-write-model-seed.json`
- `site/src/data/generated/cms-public-read-model.json`
- `site/src/data/generated/cms-migration-diff-report.json`

Prompt 53 should build the constrained editorial UI around this contract, not around raw public collection files.

Important rules to preserve:

- the public site only reads the generated published snapshot
- only `published` revisions enter the public read model
- draft/review/scheduled content stays out of the public origin
- write-side revision metadata must stay admin-only
- layout, routing, styling, and executable behavior remain outside editor control

UI work for Prompt 53 should attach to:

- document records
- revision state
- diffability
- approval and publish transitions
- field selectors and mutation primitives from the schema catalog

Do not bypass the generated read-model boundary when adding preview or editorial components.
