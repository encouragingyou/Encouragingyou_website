# Handover: Prompt 06 to Prompt 07

Prompt 06 is complete.

## What changed

- Turned `site/` into a real Astro app root with:
  - `astro.config.mjs`
  - `tsconfig.json`
  - runtime dependencies in `package.json`
  - `@astrojs/check` and TypeScript 5.x compatibility for `astro check`
- Added a legacy public-asset sync path via `site/scripts/sync-legacy-public.mjs`.
- Added a machine-readable route/asset bridge manifest in `site/src/lib/legacy/bridge-manifest.js`.
- Added a synced-public bridge for legacy prototype pages via `site/scripts/sync-legacy-public.mjs` and `site/src/lib/legacy/bridge-manifest.js`.
- Added Astro-owned minimum viable routes for:
  - `/programmes/`
  - `/programmes/[slug]/`
  - `/events-updates/`
  - `/contact/`
  - `/volunteer/`
  - `/partner/`
  - `/cookies/`
  - `/accessibility/`
  - `/terms/`
- Repointed Playwright to the Astro build-and-preview runtime instead of the prototype static preview.
- Added migration docs:
  - `docs/architecture/06-stack-realization.md`
  - `docs/migration/06-legacy-bridge-contract.md`
  - `docs/migration/06-route-parity-matrix.json`
  - `docs/migration/06-cutover-checklist.md`
  - `docs/migration/06-risk-register.md`

## What is now proven

- The new production app exists as a real executable Astro codebase.
- Browser QA now enters through the Astro runtime.
- Legacy routes can stay live during cutover without becoming a second app.
- Structured content now powers more than just the homepage.

## Primary files for prompt 07

- `site/package.json`
- `site/astro.config.mjs`
- `site/tsconfig.json`
- `site/src/lib/legacy/bridge-manifest.js`
- `site/src/pages/programmes/**`
- `site/src/pages/contact/index.astro`
- `site/scripts/sync-legacy-public.mjs`
- `site/scripts/generate-route-parity-matrix.mjs`
- `docs/migration/06-route-parity-matrix.json`

## Known limitations

- Bridged legacy pages still depend on prototype CSS and JS.
- Several new Astro-owned routes are intentionally minimum viable placeholders, not final designs.
- `preview:prototype` still exists for direct legacy comparison, but Playwright is no longer using it by default.
- Plain `npm` still defaults into workspace mode in this environment, so verified commands used `--workspaces=false`.

Prompt 07 should formalize developer tooling around this now-real app root instead of re-scaffolding the runtime again.
