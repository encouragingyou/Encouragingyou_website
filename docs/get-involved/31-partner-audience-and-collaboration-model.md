# Prompt 31 - Partner Audience and Collaboration Model

The partner route now distinguishes two layers of truth.

## Audience layer

The public page explicitly supports:

- schools, colleges, and referral-minded professionals
- community organisations, faith groups, and local networks
- local businesses, funders, venue partners, and practical supporters

This is modeled in `routePages.partner.audienceSection`, not inferred from free copy.

## Collaboration layer

The page also defines three public collaboration modes in `routePages.partner.pathwaysSection`:

- referral and signposting conversations
- joint delivery, events, and community collaboration
- sponsorship, venue, in-kind, or practical local support

Each mode uses the shared `InvolvementRoleCard` pattern so the page can explain:

- what kind of conversation the route is for
- what the first message should contain
- where a visitor should go next

## Credibility model

Launch trust comes from what is already public:

- youth-led and community-rooted positioning
- visible programmes and live session routes
- privacy and safeguarding links close to the enquiry path
- a Rochdale-centred service-area description with no invented coverage promise

## Future-proof proof model

`routePages.partner.proofBoundary` now governs three buckets:

- safe to publish now
- still waiting for approval
- held back until verified

That leaves a clean path for later partner logos, case studies, testimonials, or sector-specific proof without reopening the route architecture.
