# Prompt 23 - Audience Expansion And Wayfinding Model

## Direct Audiences

The route now speaks directly to four visitor shapes:

1. older adults or isolated community members
   - need connection, company, or a first support conversation
   - should be routed to `Contact the team`
2. carers, relatives, and referrers
   - may need to ask before a visit is realistic
   - should also be routed to `Contact the team`
3. people looking for something live and already published
   - should be routed to `See current sessions`
4. supporters, volunteers, and potential collaborators
   - should be routed to `Get involved`

## Why This Matters

Before Prompt 23, the programme record existed but the route did not actively separate those intents.

That created four risks:

- broader community visitors could assume the site was youth-only
- supporters could mistake the route for a volunteering page
- visitors needing a live timetable could assume the page was missing data
- cross-age language could drift toward implied care services

## Wayfinding Rules

The route now follows these rules:

- use contact for support, connection, fit, or asking on someone else's behalf
- use sessions for the clearest currently published live activity
- use get involved for support-the-work intents
- use safeguarding for concerns rather than general support messages

## Boundary Rules

The page copy now actively prevents four confusion modes:

- not an event listing
- not a regulated care-service route
- not a youth-club detail page
- not a volunteer or partner detail page

## Shared Runtime Outcome

Because `audienceRoutesSection` is now part of the family, the same structure can be reused later when another route needs:

- multiple audience entry points
- explicit path separation
- low-friction signposting without duplicate page architecture
